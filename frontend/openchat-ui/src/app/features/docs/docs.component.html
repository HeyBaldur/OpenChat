Architecture & Documentation

OpenChatAi · Technical reference for developers

01

Overview

OpenChatAi is a full-stack AI chat application. Users chat in real time with a local LLM (Ollama / llama3) through a .NET 9 streaming API. Conversations and token usage are persisted in MongoDB. The Angular 21 frontend renders streamed responses token-by-token using Server-Sent Events and requestAnimationFrame batching at 60 fps.

Note on internet connectivity

Normal chat is fully offline. Two scenarios require internet: (1) the first-time download of an Ollama model (e.g. ollama pull qwen2.5), and (2) when the AI uses the fetch_url tool to retrieve content from your allowed domains. In both cases, your messages and conversation history never leave your machine — only the LLM's outbound web fetches touch external servers.

The default model and Ollama connection are set in appsettings.json. Each conversation can override the model independently via the ModelSelectorComponent in the chat UI — stored per-conversation in MongoDB and updated via PUT /conversation/:id/model.

  2. 2 Open appsettings.json and set "Model": "qwen2.5:7b" as the new default
  3. 3 Restart the API — press R in dev.ps1 or run dotnet watch run
  4. 4 Or switch model per-conversation via the model selector in the chat UI — no restart required.

AI Behavior & Hallucinations

Understanding how the LLM generates responses and what to do when it behaves unexpectedly.

How the System Prompt works

Every request to Ollama begins with a system message — a set of instructions that defines the assistant's personality, scope, and behavior rules. This is injected before the conversation history on every single call. The model never sees a request without these instructions.

The system prompt is defined in backend/OpenChat.API/Services/OllamaService.cs and can be overridden via appsettings.json → Ollama:SystemPrompt.

What is a hallucination?

LLMs do not "know" facts — they generate statistically likely responses based on their training data. When asked something the system prompt does not address (like "who built you?"), the model may invent a plausible-sounding answer rather than admitting it doesn't know. This is called a hallucination.

Real example from this app:

User asked: "Who is your developer?"

Model responded: "My developer is a company called Rendlement..."

"Rendlement" does not exist — the model fabricated it because the system prompt did not specify the developer.

Fix already applied

The system prompt now includes an explicit instruction that prevents the model from inventing a developer name: 

IMPORTANT: If asked who created you, who your developer is,
or who built you, say that you are a custom AI assistant.
Do not invent, guess, or name any company or individual
as your developer.
